# Chat categories

**Status:** stub, populated in Phase 4.

This document will define the taxonomy of chat states used by the classifier and routing rules. Categories will include funnel stages (cold, warm, hot, negotiating, buyer, post-purchase) and side states (timewaster, abusive, paid-and-waiting, ghosted, human-active).

Each category will specify:
- Definition and entry signals
- Exit signals (what moves a chat out of this category)
- Routing action (auto-respond / queue / ignore)
- Response latency profile
- Persona instruction overlay
